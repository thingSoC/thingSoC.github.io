thingSoC Frequently Asked Questions (FAQ)

  • Q: What is thingSoC?
  • A: thingSoC is an open source standard that defines a vendor independent socket system for the creation of new Internet Things; it addresses many of the limitations of current product offerings and standards, by adding capabilities such as automatic device discovery, automatic device configuration, self-monitoring, and self-testing.
  • Q: Is thingSoC hardware, firmware, or software?
  • A: thingSoC defines both a physical, hardware socket system for interoperable boards, as well as a firmware model for automatic device discovery, and a software API for interacting with the system.
  • Q: Why create a new standard? Why not just use Arduino, or Beaglebone, or one of the many others?
  • A: thingSoC standardizes several new capabilities that the existing ones lack, and is a completely open source standard, not tied any particular vendor implementation, CPU architecture, or IDE.
    thingSoC is also designed to be backward compatible with Arduino, rfDuino, Raspberry Pi, Beaglebone, X-Bee, and many others.
  • Q: How big are the thingSoC modules?
  • A: A "Single" thingSoC module is 38.1mm long by 38.1mm wide (1.5 inches x 1.5 inches).
    A "Double" thingSoC module is 79.2mm long by 38.1mm wide (3.12 inches x 1.5 inches).
    A "Triple" thingSoC module is 114.8mm long by 38.1mm wide (4.52 inches x 1.5 inches).
