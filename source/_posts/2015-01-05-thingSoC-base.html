thingSoC is an open source project that defines a vendor independent system socket standard for the creation of new devices for the Internet of Things. thingSoC addresses many of the limitations of current product offerings; such as automatic device discovery, automatic device configuration, self-monitoring and self-testing capabilities.

thingSoC can support many existing hardware small form factors, such as Arduino, rfDuino, Raspberry Pi, Beaglbone, X-Bee, and others; but adds new features and capabilites to make building your IoT devices easier, with a more reliable and portable final product.

thingSoC provides a vendor neutral and device agnostic socket standard, using a virtual hardware definition. This means that the IoT applications you develop can be easily migrated without having to re-write your software.

thingSoC firmware is based on the Reference System Virtual Platform (RSVP-SIS), a virtual platform definition that supports a number of different System-on-Chip platforms.

{{page.product_slug}} Features:

  • Auto-Discovery of attached peripherals and radios
  • Auto-Configuration of attached peripherals
  • Auto-monitoring of attached peripherals
  • Standard metadata store for configuration information
  • An open-source hardware standard (OSHW), with no licensing costs

We are currently working with several alpha test customers, and have the beta release planned for May 2015.
