thingSoC Firmware Architecture

  • Q: What is thingSoC firmware?
  • All thingSoC Compliant boards contain an I2C EEPROM Memory device (minimum 32Kbits) which contains data that identifies the board type, it's serial number, pin multiplexing and setup information. THis allows a thingSoC Compliant system to automatically discover what boards have been installed into a system, and to configure itself properly to communicate with them.
  • Q: Is the thingSoC firmware autodiscovery mechanism compatible with the Beaglebone Cape Manager?
  • Yes. thingSoC Compliant devices implement the thingSoC EEPROM based autodiscovery mechanism, which is compatible with the Linux 3.8 Kernal and Device Tree Overlays. The thingSoC EEPROM based autodiscovery mechanism also implements additional features, however it is backward compatible with the Linux Beaglebone Cape Manager.
